# 全量微调与高效微调

## 一句话区分

同样是微调,区别在于**改多少参数**:

- **全量微调 (Full Fine-tuning)**:把模型的**所有**参数都拿来训练、更新。
- **高效微调 (PEFT, Parameter-Efficient Fine-Tuning)**:**绝大部分参数冻结不动**,只训练一小部分(新增的或选中的)。

## 为什么会分出这两条路

一个 7B 模型有 70 亿个参数,70B 模型有 700 亿。全量微调意味着每一步训练都要计算并更新**全部**这些参数的梯度。

这带来三个现实问题:

1. **显存爆炸**:训练时不只存参数,还要存梯度、优化器状态(Adam 要存动量),实际显存需求是参数本身的好几倍。全量微调一个 7B 模型轻松吃掉 200GB+ 显存,普通人根本跑不动。
2. **算力/时间贵**:动的参数越多,训练越慢、成本越高。
3. **灾难性遗忘风险高**:全部参数都在变,很容易把预训练学到的通用能力一起改坏(见 [[05 灾难性遗忘|灾难性遗忘]])。

高效微调就是为了解决这些问题诞生的:**既然全改代价这么大,能不能只动一小撮关键参数,就达到接近的效果?** 答案是可以——这就是 PEFT 的核心思想。

> [!key-insight] 一个反直觉的事实
> 高效微调只训练 **不到 1%** 的参数,效果却常常逼近全量微调。原因是:适配一个下游任务,需要的"改动量"其实很小(低秩假设),不需要动整个模型。详见 [[03 LoRA 与 QLoRA|LoRA 与 QLoRA]]。

## 直观类比

| | 全量微调 | 高效微调 |
|---|---|---|
| 比喻 | 重新装修整栋房子 | 只换关键家具/加个隔断 |
| 改动范围 | 推倒重来,处处都动 | 主体不动,局部调整 |
| 成本 | 高(钱、时间、风险) | 低 |
| 翻车风险 | 高(可能把好的也拆了) | 低(主体结构受保护) |

## 完整对比

| 维度 | 全量微调 | 高效微调 (PEFT) |
|------|------|------|
| 更新参数 | 全部(100%) | 极少(常 <1%) |
| 显存/算力 | 海量 | 轻量 |
| 训练速度 | 慢 | 快 |
| 能力改造深度 | 深、彻底 | "四两拨千斤",多数任务够用 |
| 灾难性遗忘风险 | 高 | 低(主体权重冻结) |
| 多任务部署 | 每个任务存一份完整模型 | 共享底座 + 切换小适配器,省空间 |
| 适用场景 | 任务与预训练差异极大、精度要求极高、资源充足 | 工业界主流,资源受限的绝大多数场景 |

## PEFT 的三大类(怎么"只动一小撮")

PEFT 不是单一方法,而是一类思路,主要分三种实现:

1. **增加额外参数 (Addition-based)**:原模型不动,在中间**插入**新的小模块来训练。
   - 代表:Adapter(层间插适配器)、Prefix/Prompt Tuning(在输入端加可训练前缀)
2. **选取部分参数更新 (Selection-based)**:从原模型里**挑**一部分参数来训,其余冻结。
   - 代表:层级冻结(只训顶层)、BitFit(只训偏置项)
3. **重参数化 (Reparametrization-based)**:用一个**低秩的小矩阵**来表示参数的改动量。
   - 代表:[[03 LoRA 与 QLoRA|LoRA]] —— 目前最主流

## 高效微调用在哪(四大应用场景)

实际大模型应用中,高效微调主要用于以下四个方面。先看完整说明,再看下方提炼的速查表。

- **对话风格微调**:高效微调可以根据特定需求调整模型的对话风格。例如针对客服系统、虚拟助理等场景,模型可以通过微调来适应不同的**语气、礼貌程度**或**回答方式**,从而在与用户互动时提供更符合要求的对话体验。通过微调少量的参数(例如对话生成的策略、情感表达等),可以使模型表现出更具针对性和个性化的风格。

- **知识灌注**:知识灌注是指将外部知识或领域特定的信息快速集成到已有的预训练模型中。通过高效微调,模型可以更好地学习新领域的专有知识,而无需重新从头开始训练。例如对于法律、医疗等专业领域,可以使用少量的标注数据对预训练模型进行微调,帮助模型理解特定行业的**术语、规则和知识**,进而提升专业领域的问答能力。

- **推理能力提升**:高效微调还可以用于提升大模型的推理能力,尤其是在处理更复杂推理任务时。通过微调,模型能够更高效地**理解长文本、推理隐含信息,或从数据中提取逻辑关系**,进而在多轮推理任务中提供更准确的答案。这种微调方式可以帮助模型在解答复杂问题时提高推理准确性并减少错误。

- **Agent 能力(Function Calling 能力、或者 MCP 能力)提升**:在多任务协作或功能调用场景中,高效微调能够显著提升模型的 **Agent 能力**,使得模型能够有效地与其他系统进行交互、调用外部 API 或执行特定任务。通过针对性微调,模型可以学会更精准的**功能调用策略、参数解析和操作指令**,从而在自动化服务、智能助手或机器人控制等领域表现得更加高效和智能。

### 速查表(提炼)

| 场景 | 解决什么 | 例子 |
|---|---|---|
| **① 对话风格微调** | 调整模型的**语气、礼貌程度、回答方式**,让对话更具针对性和个性化 | 客服系统、虚拟助理:微调少量参数(对话策略、情感表达)就能换一种说话风格 |
| **② 知识灌注** | 把**外部/领域专有知识**快速集成进已有模型,无需从头训练 | 法律、医疗等专业领域:用少量标注数据让模型懂行业术语、规则,提升专业问答能力 |
| **③ 推理能力提升** | 增强模型**理解长文本、推理隐含信息、提取逻辑关系**的能力 | 多轮推理任务:微调后在解答复杂问题时推理更准、错误更少 |
| **④ Agent 能力提升** | 提升 **Function Calling / MCP** 等工具调用能力,让模型会与外部系统交互、调用外部 API、执行特定任务 | 多任务协作/功能调用场景:微调后模型学会更精准的**功能调用策略、参数解析和操作指令**,在自动化服务、智能助手、机器人控制等领域更高效智能 |

> [!key-insight] 怎么对应到前面的方法
> - **①对话风格 / ③推理** 更偏"调行为模式" → 调 Attention 层(`q/v` 起步),数据用风格/推理样本。
> - **②知识灌注** 更偏"补事实" → 可加 FFN 层(`gate/up/down`),数据用领域语料。(对应 [[03 LoRA 与 QLoRA|LoRA 与 QLoRA]] 里 target_modules 的搭配)
> - **④Agent** 本质也是一种指令微调:用大量"任务→正确的工具调用"样本训练(对应 [[04 SFT 与指令微调|SFT 与指令微调]])。

## 一个具体场景:多任务部署省钱

假设你的公司要为 5 个业务(客服、法律、营销、代码、翻译)各做一个专用模型:

- **全量微调**:要存 5 份完整的 7B 模型 ≈ 5 × 14GB = 70GB,而且每份都要单独训练。
- **LoRA(高效微调)**:存 1 份共享底座(14GB)+ 5 个小适配器(每个几十 MB)。

> [!key-insight] 这就是工业界几乎都用 PEFT 的原因
> 一个底座 + 多个可插拔适配器,训练快、存储省、上线灵活。需要哪个业务就挂哪个适配器。

## 怎么选

- 默认、绝大多数情况 → **高效微调(优先 LoRA / QLoRA)**
- 只有当任务与预训练差异极大、且对精度有极致要求、资源也充足时 → 考虑**全量微调**
