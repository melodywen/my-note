# 01 Qwen3 混合推理模型微调数据集准备

> 本篇是「Qwen3 微调经验总结」系列开篇,聚焦一个**最容易踩坑、又最关键**的环节:**给 Qwen3 准备微调数据集时,怎么既提升它某项专长(比如数学),又不把它原本的"会聊天、会推理"能力给搞丢**。

## 1. 核心命题:别让微调把模型练"偏科"了

Qwen3 是个 **混合推理模型**——它既能像普通模型一样正常对话(直接给答案),又能在需要时进入"思考模式"(`<think>` 里一步步推理再作答)。

> [!important] <font color="red">直接拿单一领域数据(比如只喂数学题)去微调,会出大问题</font>
> 如果你只用数学推理数据去训 Qwen3,它的数学是变强了,但会**忘掉**怎么正常聊天、怎么应对非推理任务——这就是 [[05 灾难性遗忘]]。模型变成了一个"只会做数学题、不会好好说话"的偏科生。

所以正确做法不是"喂什么强什么",而是要**配比**:在提升目标能力的同时,掺入足够的通用数据来"保住"原有能力。

## 2. 解法:两类数据集"拼接 + 配比"

视频里给的经验做法,是把**两类数据**混在一起训练:

| 数据集 | 类型 | 作用 | 地址 |
|---|---|---|---|
| **FineTome-100k** | 普通对话数据 | 保住**非推理**的日常对话能力 | huggingface.co/datasets/mlabonne/FineTome-100k |
| **OpenMathReasoning-mini** | 带推理字段的数学数据(COT) | 提升**数学/推理**能力 | huggingface.co/datasets/unsloth/OpenMathReasoning-mini |

> [!key-insight] 一句话理解配比
> - **OpenMathReasoning**(数学 COT)= 让模型变聪明的"专项训练"
> - **FineTome**(普通对话)= 防止它"练废了不会说话"的"通识保底"
> - 两者**拼接**起来一起训,模型才能"既会算题,又会聊天"。

## 3. 关键参数是"配比",而且要边训边调

> [!warning] 配比不是一次定死,要在训练过程中持续调整
> 视频原文强调:**"需要在持续微调训练过程中不断调整 COT 数学数据集和普通文本问答数据集之间的配比"**。
>
> - 数学数据放太多 → 数学强了,但对话/通用能力退化(偏科)
> - 普通对话放太多 → 对话保住了,但数学没怎么提升(没练到点上)
> - 要找到那个**平衡点**:数学能力上去了,混合推理性能也没掉。

这就像配营养餐:不能光吃肉(数学),也不能光吃菜(对话),要按比例搭配,还要根据"体检结果"(评测)动态调整。

## 4. 什么是"带推理字段"的数据集?

普通对话数据长这样(直接给答案):

```json
{
  "messages": [
    {"role": "user", "content": "1+1等于几?"},
    {"role": "assistant", "content": "等于2。"}
  ]
}
```

而 **带推理字段(COT)** 的数学数据,答案里包含**思考过程**:

```json
{
  "messages": [
    {"role": "user", "content": "一个数加上3等于7,这个数是几?"},
    {"role": "assistant", "content": "<think>设这个数为x,x+3=7,所以x=7-3=4</think>\n\n这个数是4。"}
  ]
}
```

> [!key-insight] 为什么要带 `<think>` 推理过程?
> 因为 Qwen3 的"混合推理"能力,靠的就是这种**先想后答**的模式。用带推理过程的数据训练,才能强化/保留它在 `<think>` 里一步步推理的本事。如果只给"直接答案",反而会削弱它的推理习惯。
>
> COT = **C**hain **o**f **T**hought,思维链,即把推理过程一步步写出来。

## 5. 给你(Mac 入门)的落地建议

> [!important] 入门阶段不必上来就追求"完美配比"
> - 你现在用 Qwen3-0.6B + Mac 验证流程,**重点是把混合数据集跑通**,理解"为什么要掺通用数据"这件事。
> - 先各取一小部分(比如数学 + 对话各几千条)拼起来跑,观察微调后模型**还会不会正常聊天**——这就是验证"有没有偏科"。
> - 真正的配比调优(7:3、5:5 反复试)是有了评测体系之后的事,见 [[模型评测]]。

> [!warning] 数据集格式要先转成 ms-SWIFT 认的格式
> HuggingFace 上的数据集字段不一定直接能用,通常要转成 ms-SWIFT 的 `messages` 格式。数据集准备和格式见 [[07 微调数据集的创建原理]]。
