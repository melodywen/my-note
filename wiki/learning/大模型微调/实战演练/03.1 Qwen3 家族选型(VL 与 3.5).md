# 03.1 Qwen3 家族怎么选:带不带 VL、3 还是 3.5

## 1. 带 VL vs 不带 VL —— 核心区别:能不能"看图/看视频"

> [!key-insight] 一句话:**VL 版 = 文本版 + 一双"眼睛"(ViT 视觉编码器)**
> `VL = Vision Language(视觉语言)`。不带 VL 是**纯文本大语言模型**;带 VL 是**多模态模型**,在文字之外还能理解图像、视频。

| 维度 | Qwen3(不带 VL) | Qwen3-VL(带 VL) |
|---|---|---|
| **模型类型** | 纯文本 LLM | 多模态(图像 + 视频 + 文本) |
| **能干嘛** | 对话、写作、推理、代码 | 上述 + **图文问答、视频理解、目标检测(grounding)** |
| **数据集字段** | 只有 `messages` 文本 | 多出 `images`/`videos` 字段,正文里用 `<image>`/`<video>` 占位 |
| **多出的结构** | 无 | **ViT 视觉编码器** + **aligner 对齐层**(把图像特征转成模型能懂的向量) |
| **特有训练参数** | 无 | `--freeze_vit true`、`--freeze_aligner true`(冻结视觉部分省显存)<br>环境变量 `IMAGE_MAX_TOKEN_NUM` / `VIDEO_MAX_TOKEN_NUM` / `FPS_MAX_FRAMES`(控制图/视频吃多少 token) |
| **显存** | 8B 约 22GB | 4B 就要 2×21GiB(多了 ViT,还要处理图像) |
| **示例模型** | `Qwen3-8B`、`Qwen3-32B` | `Qwen3-VL-4B-Instruct`(Dense)、`Qwen3-VL-30B-A3B-Instruct`(MoE) |

**VL 数据集长这样**(比纯文本多了 `images` 字段和 `<image>` 占位符):
```json
{"messages": [{"role": "user", "content": "<image><image>两张图片有什么区别"},
              {"role": "assistant", "content": "前一张是小猫,后一张是小狗"}],
 "images": ["/xxx/a.jpg", "/xxx/b.png"]}
```

**VL 训练命令的差异**(在普通 `swift sft` 基础上多了视觉相关设置):
```bash
IMAGE_MAX_TOKEN_NUM=1024 VIDEO_MAX_TOKEN_NUM=128 FPS_MAX_FRAMES=16 \
swift sft \
    --model Qwen/Qwen3-VL-4B-Instruct \
    --tuner_type lora \
    --freeze_vit true \            # 冻结"眼睛",只调语言部分,省显存
    --freeze_aligner true \
    --attn_impl flash_attn \
    ...
```

> 为什么常冻结 ViT?视觉编码器已在海量图文上预训练好,**通常不用动**;只微调语言部分既省显存又更稳,这正是 `freeze_vit`/`freeze_aligner` 默认推荐 true 的原因。

## 2. Qwen3 vs Qwen3.5 —— 核心区别:架构升级 + 多模态原生融合

> [!key-insight] 一句话:**3.5 = 3 的"架构升级版"**
> 换了更高效的**混合注意力(GatedDeltaNet / GDN)**,把多模态**原生融进一个模型**,思考模式做成可显式开关。

| 维度 | Qwen3 | Qwen3.5 |
|---|---|---|
| **注意力架构** | 标准 full attention | **混合注意力**:linear attention(GDN)+ full attention,长文本更省更快 |
| **多模态** | 文本版与 VL 版**分两套** | **原生融合**,一个模型同时吃文本 + 图像 + 视频 |
| **思考模式** | 有 thinking,用 `--response_prefix` 控制 | 升级为**混合思考**,新增 `--add_non_thinking_prefix`、`--enable_thinking` 显式开关 |
| **后端依赖** | 常规 | 需 `flash-linear-attention>=0.4.2`(GDN 算子);对 transformers 版本敏感(5.2 与 vllm 冲突、5.3 有视频训练问题) |
| **推荐参数** | `--loss_scale ignore_empty_think` | 同上 + `--group_by_length true`;MoE 用 `--experts_impl grouped_mm` |
| **示例模型** | `Qwen3-8B` 等 | `Qwen3.5-4B`(Dense)、`Qwen3.5-35B-A3B`(MoE) |

> [!warning] 三者通用的两个"思考模式"参数(Qwen3 系列都涉及)
> - `--loss_scale ignore_empty_think`:**忽略空思维链的损失**,防止微调后丢掉推理能力。
> - 数据里加 `/no_think` 标记 / 混入非思维链数据,同样是为**缓解推理能力丢失**。
> 这是 Qwen3 系列(含 VL、3.5)做 SFT 时的高频注意点。

## 3. 选型速查

> [!key-insight] 我该用哪个?
> - **只处理文字**(对话/写作/代码) → **Qwen3**(不带 VL),最简单。
> - **要看图/看视频**(图文问答、视频理解、检测) → **Qwen3-VL**。
> - **想要最新架构 + 长文本更省 + 文本多模态一体** → **Qwen3.5**(注意依赖更挑,需 FLA 和特定 transformers 版本)。
> - **Mac 练手**:仍建议用小号纯文本模型(如 `Qwen3-0.6B`)跑通流程;VL / 3.5 的环境依赖在 Mac 上更容易踩坑,验证流程优先上 CUDA。
