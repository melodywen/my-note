# 03.2 显卡 / 算力监控工具

> 本页从 [[03 ms-SWIFT 测试运行示例]] 拆出。微调跑起来后,得**盯住显卡**:显存够不够、利用率高不高、温度功耗正不正常。一句话记住:**`nvidia-smi` 是官方自带的"体检表"(看一眼),`nvitop` 是彩色动态的"仪表盘"(实时盯)。** Mac 没有 N 卡,另有自己的看法,见末尾。

## 1. nvidia-smi —— NVIDIA 官方自带,装了驱动就有

`nvidia-smi`(NVIDIA System Management Interface)随显卡驱动安装,**无需额外装**,是最基础、最通用的查看方式。

```bash
nvidia-smi              # 打印一次当前快照:显存/利用率/温度/功耗/进程
nvidia-smi -l           # loop:每秒刷新一次(默认间隔,持续打印)
nvidia-smi -l 1         # 每 1 秒刷新一次(显式指定间隔)
watch -n 1 nvidia-smi   # 用 watch 每 1 秒刷新(画面原地更新,不滚屏,更清爽)
nvidia-smi topo -m      # 打印多卡拓扑(GPU 间通过 NVLink/PCIe 怎么连),多卡训练排查通信用
```

> [!key-insight] `nvidia-smi -l` 和 `watch -n 1 nvidia-smi` 都能"每秒看一次",区别在哪?
> - `nvidia-smi -l 1`:由 nvidia-smi **自己循环**,结果**一屏一屏往下滚**(历史留在屏幕上)。
> - `watch -n 1 nvidia-smi`:由 `watch` 反复执行命令,画面**原地刷新**(只看最新一屏,不滚屏)。
> 想盯实时状态用 `watch` 版更舒服;想留记录看变化用 `-l` 版。

**怎么读这张表**(训练时重点看三个数):
| 看哪 | 字段 | 说明 |
|---|---|---|
| **显存** | `Memory-Usage`(如 `13000MiB / 24576MiB`) | 已用 / 总量。爆显存(OOM)就调小 batch / max_length |
| **利用率** | `GPU-Util`(如 `98%`) | GPU 算力被吃掉多少。训练时应接近 100%;若一直很低,多半卡在数据加载或 CPU |
| **进程** | 下方 `Processes` 列表 | 哪个 PID 在占卡、占了多少显存,排查"谁占着卡不放"用 |

## 2. nvitop —— 彩色交互式"仪表盘"(更好用,需 pip 装)

`nvidia-smi` 信息全但是是**静态文本**;`nvitop` 是第三方工具,把这些信息做成**彩色、动态、可交互**的界面(类似 `htop` 之于 `top`),实时看曲线、按进程排序、甚至直接 kill 进程。

```bash
pip install nvitop      # 安装(一次即可)

nvitop -m auto          # 自动模式:按终端大小自适应布局(默认推荐)
nvitop -m compact       # 紧凑模式:信息密,占地方小
nvitop -m full          # 完整模式:显示最全,信息最多
```

> [!key-insight] 什么时候用 nvitop 而不是 nvidia-smi?
> - **快速瞄一眼** → `nvidia-smi`,不用装、随手就有。
> - **长时间盯训练 / 多卡 / 想看利用率曲线和趋势** → `nvitop`,彩色直观、能交互,看着不累。
> 两者不冲突,nvitop 底层也是读同一套 NVIDIA 接口的数据。

## 3. 其他可选监控手段

- **`gpustat`**:`pip install gpustat`,比 nvidia-smi 更简洁的一行式概览,适合脚本里快速打点。
- **`wandb`**:训练过程在线记录工具,除了 loss 曲线,也能**自动记录 GPU 利用率/显存/温度**,适合复盘整段训练。见 [[01 Qwen3 高效微调环境准备]] 里的工具四件套。
- **DCGM / Prometheus + Grafana**:集群级监控,生产环境多卡长期监控才需要,练手用不上。

## 4. Mac(Apple Silicon / MPS)没有 nvidia-smi 怎么办?

> [!warning] 上面这些 `nvidia-smi` / `nvitop` 在 Mac 上**都用不了**(它们只认 NVIDIA 卡)
> Mac 用的是苹果自家 GPU(走 MPS),没有 N 卡那套工具。替代方案:
> - **活动监视器(Activity Monitor)** → 「窗口 → GPU 历史记录」:看 GPU 占用曲线。
> - **`sudo powermetrics --samplers gpu_power`**:命令行看 GPU 功耗/占用。
> - **统一内存要一起看**:Mac 是 CPU/GPU 共享内存,跑模型主要盯**内存压力**(活动监视器「内存」页),内存吃满会严重掉速甚至崩。
>
> 这也呼应 [[03 ms-SWIFT 测试运行示例]] 的结论:**Mac 适合验证流程,真正训练 + 监控还是上 CUDA。**
