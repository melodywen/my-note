# 四大微调框架概述

> 本页是「微调框架」系列的总览。先讲**为什么用现成框架**,再横向对比 **Unsloth / LLaMA-Factory / ms-SWIFT / ColossalAI** 四套主流工具,帮你按场景选型。每套框架后续各开一篇细讲。

## 一句话定位

入门学习大模型微调时,**优先推荐封装层次较高的四套工具**:

- **Unsloth**
- **LLaMA-Factory**
- **ms-SWIFT**
- **ColossalAI**

除此之外也能借助更底层的库(如 [[03 LoRA 与 QLoRA|peft]]、`transformers`、`trl`)手写实现高效微调。但对初学者来说,**先用现成工具跑通一轮**,再往下钻原理,是最高效的路径。

> [!key-insight] 为什么先用框架,而不是从头写
> 微调的"脏活"(数据加载、分布式、量化、断点续训、显存优化、各模型适配)框架都替你封装好了。你只要准备数据 + 改配置就能开训。**框架不是偷懒,而是把精力留给真正重要的环节:数据质量和超参。**

## 框架分两个层次

| 层次 | 代表 | 你需要做什么 |
|---|---|---|
| **高层封装(推荐入门)** | Unsloth、LLaMA-Factory、ms-SWIFT、ColossalAI | 准备数据 + 写配置 / 点 UI,开箱即用 |
| **底层库(进阶/定制)** | `peft`、`transformers`、`trl`、`bitsandbytes` | 自己拼训练循环、自己管显存,灵活但门槛高 |

四套高层工具内部,其实也大多是**调用底层的 peft / transformers** 来实现 LoRA、QLoRA —— 框架是"壳",底层库是"芯"。

## 四大框架横向对比

| 框架                | 一句话定位    | 最大优势                                               | 主要短板                    | 适合谁                     |
| ----------------- | -------- | -------------------------------------------------- | ----------------------- | ----------------------- |
| **Unsloth**       | 单卡极致提速器  | **速度快 2~5 倍、显存省 ~70%**,手写 GPU 内核优化;API 简洁、新手友好     | 多卡/分布式支持弱,模型覆盖偏主流       | 单卡(消费级显卡)、资源受限、快速迭代     |
| **LLaMA-Factory** | 全能型瑞士军刀  | **支持 100+ 模型**、带 Web UI(零代码)、方法最全(SFT/DPO/PPO/多模态) | 速度不如 Unsloth,全参微调显存要求高  | 多硬件环境、复杂任务、想要图形界面       |
| **ms-SWIFT**      | 魔搭生态全家桶  | **支持 450~600+ 模型 + 上百多模态模型**;训练→推理→评测→量化→部署一条龙     | 强依赖 ModelScope 生态       | 国产模型(Qwen 等)、多模态、要端到端流程 |
| **ColossalAI**    | 大规模分布式引擎 | **四大并行(数据/张量/流水线/ZeRO)**,专攻千亿级超大模型多机多卡训练           | 偏底层、配置复杂,小模型/单卡用它是杀鸡用牛刀 | 超大模型、多机多卡集群、预训练级别任务     |

> [!key-insight] 一句话记住四者差异
> **Unsloth 比快,LLaMA-Factory 比全(还有 UI),ms-SWIFT 比生态(模型多、能部署),ColossalAI 比规模(多机多卡训巨型模型)。**

## 选型决策树

```text
你的场景是?
├─ 单张消费级显卡,想最快跑通          → Unsloth
├─ 想要图形界面 / 方法最全 / 多模型对比 → LLaMA-Factory
├─ 主用国产模型(Qwen)或需要端到端部署 → ms-SWIFT
└─ 千亿参数 / 多机多卡集群 / 预训练级别 → ColossalAI
```

## 关键维度速查

| 维度 | Unsloth | LLaMA-Factory | ms-SWIFT | ColossalAI |
|---|---|---|---|---|
| 上手难度 | 低 ⭐ | 中(有 UI) | 中 | 高 |
| 单卡效率 | 最高 ⭐ | 中 | 中 | 一般 |
| 分布式/多卡 | 弱 | 支持 | 支持 | **最强** ⭐ |
| 模型覆盖 | 主流 | 100+ | 450~600+ ⭐ | 通用 |
| 微调方法 | LoRA/QLoRA 为主 | 最全(含 RLHF/DPO) | 全(含人类对齐) | 全参/并行训练 |
| 图形界面 | 无 | **有 Gradio UI** ⭐ | 命令行为主 | 无 |
| 典型场景 | 个人/快速迭代 | 复杂任务/教学 | 国产生态/部署 | 超大规模训练 |

## 给初学者的建议路线

1. **先 Unsloth 或 LLaMA-Factory** 跑通你的第一个 [[04 SFT 与指令微调|SFT]] / [[03 LoRA 与 QLoRA|LoRA]] 任务 —— 门槛最低、最快见效。
2. 想可视化对比、试不同方法 → **LLaMA-Factory** 的 UI 最舒服。
3. 用 Qwen 等国产模型、需要训完直接部署 → **ms-SWIFT**。
4. 等真要训超大模型、上集群了,再学 **ColossalAI**。

> [!gap] 性能数字(2.5~10 倍提速、显存节省比例)来自厂商与第三方测评,会随版本、模型、硬件变化,选型时以你自己环境的实测为准。

## 注一:强化学习训练,另推荐专门框架

四大框架主攻 [[04 SFT 与指令微调|SFT]] / [[03 LoRA 与 QLoRA|PEFT]] 这条线。**如果你要做的是强化学习训练(RLHF/RLAIF 那一套),推荐用专门的 RL 框架:**

- **veRL**
- **OpenRLHF**

它们针对 RL 训练(经验采集、奖励模型、PPO/GRPO 等)做了专门优化,比通用微调框架更顺手。

## 注二:其他更底层的微调框架

四套高层工具之外,还有一批**更底层 / 更专门**的库,了解它们的定位有助于看懂整个生态:

| 框架 | 优势 | 适用场景 |
|---|---|---|
| **Hugging Face** | 高度兼容、易用、文档丰富 | 一般 NLP 任务,模型选择丰富 |
| **LoRA** | 显存节省,减少微调计算量 | 显存有限的设备,微调大规模模型 |
| **PEFT** | 高效微调,低计算开销 | 资源有限的环境,大规模预训练模型微调 |
| **DeepSpeed** | 大规模分布式训练,显存优化 | 超大规模训练,多卡分布式训练 |
| **AdapterHub** | 低资源消耗,快速微调 | 多任务微调,资源有限的环境 |
| **Alpaca-LoRA** | 生成任务优化,结合 LoRA 技术 | 对话生成、文本生成 |
| **FastChat** | 对话系统微调,快速集成 | 对话生成任务,尤其是对 ChatGPT 等模型微调 |
| **FairScale** | 大规模分布式训练优化,自动化优化 | 多卡分布式训练,大规模模型 |

> [!key-insight] 高层框架 vs 这些底层库的关系
> 上面这些大多是**底层"芯"**:四大高层框架(尤其 [[04 ms-SWIFT|ms-SWIFT]]、[[03 LLaMA-Factory|LLaMA-Factory]])内部往往就是在调用 **PEFT/LoRA(省显存)+ DeepSpeed/FairScale(分布式)** 这些库。直接用底层库灵活但门槛高;入门优先用四大高层框架。
